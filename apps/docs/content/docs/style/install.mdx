## Install with Stylus

1. Install the [Stylus extension](https://add0n.com/stylus.html) for your browser.
2. Click the badge below to add the style. Stylus will fetch the latest release automatically.

   [![install with Stylus](https://img.shields.io/badge/Install%20directly%20with-Stylus-116b59.svg?longCache=true&style=flat)](https://coolify-tweaks-api.techwithanirudh.com/release/latest/?asset=main.user.css)

3. If your Coolify instance uses a custom hostname, open Stylus, edit the styleâ€™s **Settings**, and add a wildcard host to **Custom included sites**:

   ```css
   *://your.coolify.host:8000/*
   ```

   Replace the host above with your own IP/port, then click **Save**.

4. If styling seems slow or inconsistent, enable **Instant Mode** in Stylus so the CSS injects immediately instead of waiting for the DOM to settle.

## Theme customisation with TweakCN

Coolify Tweaks delegates theming to [TweakCN](https://tweakcn.com). To apply a custom palette:

1. Browse themes or build your own at [https://tweakcn.com/editor/theme](https://tweakcn.com/editor/theme).
2. Click **Share**, copy the theme ID (e.g. `bubblegum`, `claude`, or a long hash), and append it to the installation URL.
3. Install the themed version of the style by pointing Stylus at:

   ```txt
   https://coolify-tweaks-api.techwithanirudh.com/release/latest/?theme=THEME_ID&asset=main.user.css
   ```

   Be sure the `theme` query parameter appears before `asset=main.user.css`, otherwise Stylus might treat the URL as a different style.

## Troubleshooting

- **Styles not applying?**
  - Confirm Stylus has `Accept cross-site requests` enabled.
  - Re-run the install badge to refresh the CSS.
  - Disable conflicting extensions/styles.
- **Theme looks wrong?** Refresh the page or re-save the style so the new theme loads.
- **Layout still feels tight?** Try clearing the browser cache and restarting the tab.
